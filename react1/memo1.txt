Viteとは？
・pnpm vite create
いっぱい選択肢が出てきて選んだやつ
・buildツールと開発サーバーの２つの役割がある
ブラウザはそのままではJSXやTypeScriptを読めない(純粋なjs以外受け付けない)
→それを変換してくれるのがbuildツール
・もう一つの役割である開発サーバーとは
ファイルとして開いても、importなどの便利な機能が使えない(セキュリティ上の懸念)
http://経由でアクセスしないといけない(jsの時はpackage.jsonでhttpを指定してた)

・viteを使うと、リロードしなくても画面変更をしてくれる(便利！)

<コンポーネント(TSXファイル、htmlのこと)>
ページを構成するUI部品
これから.htmlは作らず、.tsxを使う

・props = 外から値を受け取るもの、引数、親から子に値を渡す窓口
・state = 変化していく状態を管理、ローカル変数

-例題-
type HelloProps = {
    name: string;
};

export default function Hello({ name }: HelloProps) {
    return (
        <div>こんにちは、{name}さん</div>　　←子コンポーネント
    );
}


import Hello from './Hello';

createRoot(document.getElementById('root')!).render(
    <Hello name="Tom" />     ←親コンポーネント
);


・文字列以外にも渡せる
<Hello name={100} />                            数値 
<Hello name={['Tom', 'Jhon', 'Bob']} />         配列 
<Hello name={() => { console.log('Tom'); }} />  関数 
<Hello name={{ name: 'Tom', age: 30 }} />       オブジェクト
