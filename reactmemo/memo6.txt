【フロントエンド】 Reactフック(基本)
フック理解のために
1. 通常のJavaScriptの前提として「関数は呼ばれるたびにリセットされる」
        ↓
2. Reactコンポーネントも実は「ただの関数」である
        ↓
3. 「でも、ボタンを押した回数を覚えておきたい」という困りごとがある
        ↓
4. その困りごとを解決するのが「フック」である
        ↓
5. フックは大きく「状態」と「副作用」の2つに分類される
        ↓
6. フックの2つのルール(呼び出し場所・順番)
        ↓
7. 個別のフック(useState, useEffectなど)の解説に入る

・フック(引っ掛ける)
値を覚えておく、useState
Stateがないと...
function increment() {
  let count = 0;
  count = count + 1;
  return count;
}

increment(); // 1
increment(); // また1(前回の結果は覚えていない)
increment(); // また1

・フックは主に２パターン
状態の管理(useState)
副作用(useEffect) => タイマーとか決まったタイミングで何かを実行

ルール1: フックは関数コンポーネントの「一番外側」でだけ呼び出す
ルール2: フックは「関数コンポーネント」か「カスタムフック」の中でしか呼び出せない

フックはとりあえずこれだけ覚えておけ
useState,useEffect,useRef


・useEffect...決まったタイミングで何か処理を実行したい時に使うフック
Reactの外側と連携(ネットワークからデータ取得、ブラウザアクセスなど)するときに使ったりする
useEffect(() => {
  // ここに実行したい処理を書く

  return () => {
    // ここに「後片付け」の処理を書く(省略可能)
  };
}, [依存配列]);
第一引数に処理、第二引数に依存配列[]、[count]、なにも書かないなど

countが変わるたびに実行 => [count]
最初の1回だけやりたい処理 => []
再レンダリングのたびに実行 => なにも書かない

setInterval => 第一引数に処理、第二引数に秒数で第二引数後にタイマー開始、一定時間ごとに同じ処理を繰り返す
clearInterval => 引数として渡したものが消えた時にタイマー終了、これがないと描画を
終わらせてもメモリがクリアされず、自分のPCがどんどん重くなってしまう




・useLayoutEffect(ランクA)
useEffectは画面描画後に非同期で実行
useLayoutEffectは画面描画前に同期で実行される
useLayoutEffect(() => {
        sleep(2000);
        setCount(init);
    }, []);
    2秒間遅らせる処理をして、setCountに初期値を入れる
    依存配列が[]なので最初の一度だけ実行

useEffect =>　古いのが表示されてから新しいのを表示
useLayoutEffect =>最初から最新の情報を表示

・useRef...値を保存しておく、useStateと同じような役割
直接HTMLに接続する(getElementByIdみたいな)、要素オブジェクトへのアクセスが目的
あんまり使っては行けない、必要でない場合はuseStateを使うべし
useRef関数は要素へのフォーカス/スクロール/アニメーション操作のような限られた状況

useRefは、useState同様に値を保持するフックだが、決定的に異なる性質を1つ持つ
=>useRefは保持した値が変わってもコンポーネントは再描画されない(役に立つのそれ？) テスト出る

なぜuseRefは濫用しちゃいけないの？
=> Reactの根本的な設計思想に反する使い方になりがちだから
Reactはそもそも変化した部分だけ再描画するのがメインの機能だから、それに反する
いちいち再描画の命令を出さなきゃいけない

パターン1: DOM要素そのものに直接アクセスしたいとき
inputRef.current? = <input> HTMLの要素を持たせる
(使い道)動画の再生・停止を制御できる

パターン2: 「再描画を起こしたくない値」を保持したいとき
useState = 再描画される
useRef = 再描画されない
useRefを押したときは内部は変わっても再描画されないが、useStateを押した瞬間Stateの
再描画に引っ張られて再描画してくれる
=> つまりuseRefは、「再描画をまたいで値を覚えておきたいが、その値が変わったからといって
画面を更新する必要はない」というときに使う箱
const file = useRef(初期値);
file = { current: 初期値 }currentはキーのようなもの
